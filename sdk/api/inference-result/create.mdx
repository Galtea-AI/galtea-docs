---
title: 'Create Inference Result'
description: "Create a new inference result within a session"
---

## Returns

Returns an [InferenceResult](/concepts/product/inference-result) object.

## Example

```python
retrieved_context = your_retriever_function(user_query)

# Your product's actual response to the input
timeBeforeCall = datetime.now()
model_answer = your_product_call(testCase.input, retrieved_context)
timeAfterCall = datetime.now()

result = galtea.inference_results.create(
    session_id="rag-session-001",
    input={
        "query": "How do I reset my password?",
        "user_id": "user_123"
    },
    output={
        "answer": "To reset your password, follow these steps...",
        "confidence": 0.95
    },
    expected_output={
        "answer": "Password reset instructions...",
        "should_include_link": True
    },
    retrieval_context=retrieved_context,
    latency=(timeAfterCall - timeBeforeCall).total_seconds() * 1000,
)
```

## Parameters

<ParamField path="session_id" type="string" required>
  The ID of the session this inference result belongs to.
</ParamField>

<ParamField path="input" type="object" required>
  The input data that was provided to your product. This can be any JSON-serializable object.
</ParamField>

<ParamField path="output" type="object" required>
  The output generated by your product in response to the input. This can be any JSON-serializable object.
</ParamField>

<ParamField path="expected_output" type="object" optional>
  The expected or ideal output for comparison during evaluation. Useful for supervised learning scenarios.
</ParamField>

<ParamField path="retrieval_context" type="array" optional>
  Array of context documents retrieved for RAG (Retrieval-Augmented Generation) systems. Each item should contain source information and content.
</ParamField>

<ParamField path="latency" type="number" optional>
  The time taken to generate this inference result in milliseconds. Useful for performance evaluation.
</ParamField>

<ParamField path="usage_info" type="object" optional>
  Usage statistics such as token counts, API costs, model parameters, etc. Structure depends on your system.
</ParamField>
