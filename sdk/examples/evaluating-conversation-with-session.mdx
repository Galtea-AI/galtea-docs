[Evaluations](/concepts/product/evaluation) allow you to assess how well a specific [version](/concepts/product/version) of your [product](/concepts/product) performs against a set of [test cases](/concepts/product/test/case) by running individual [evaluation tasks](/concepts/product/evaluation/task).


## Creating a Session to store the conversation

To create an evaluation, you first need to create a session that will store the conversation turns and other relevant information. This session will be linked to the evaluation tasks you run later.

  ```python
  from galtea import Galtea

  # Initialize Galtea SDK
  galtea = Galtea(api_key="YOUR_API_KEY")

  # Create an evaluation
  session = galtea.sessions.create(
      id="YOUR_SESSION_ID",
      version_id="YOUR_VERSION_ID",
      test_case_id="TEST_CASE_ID",
      context="This evaluation is for testing the product's performance against a set of test cases.",
  )

  print(f"Evaluation created with ID: {evaluation.id}")
  ```
  <Info>
    An evaluation links a specific version of your product to a test. This establishes the framework for running individual evaluation tasks.
  </Info>


## Storing inference Results

Once you've created a session, you can store inference results from your product's responses to user queries. This is useful for tracking performance and analyzing how well your product handles different inputs.

```python
# Load your test cases
test_cases = galtea.test_cases.list(test_id="YOUR_TEST_ID")

# Evaluate all test cases
for test_case in test_cases:
    # store inference results
    galtea.inference_results.create(
      session_id=session.id,
      input=test_case.input,
      output=model_answer,
      retrieval_context=retrieval_component_answer,
      retrieval_context=your_retriever_function(test_case.input)
    )

print("All inference results submitted")
```

<Info>
  The `latency` and `usage_info` parameters are optional but highly recommended to be used.
  They can be used to track the performance in latency and costs of your product.
</Info>

## Running Evaluation Tasks from a session

Once you've created a session, you can run [evaluation tasks](/concepts/product/evaluation/task) using the `galtea.evaluation_tasks.create_from_session` method. This method allows you to evaluate the performance of your product against the test cases defined in the session.

```python
galtea.evaluation_tasks.create_from_session(
    session_id=session.id,
    metrics=["accuracy_v1", "relevance_v1", "context_accuracy"],
)

print("All evaluation tasks submitted")
```

<Note>
  The `metrics` parameter specifies which [metric types](/concepts/metric-type) to use for evaluating the task. You can use multiple metrics simultaneously to get different perspectives on performance.
</Note>
