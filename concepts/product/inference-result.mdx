## What is an Inference Result?

An inference result in Galtea represents a single output generated by your AI model or system. It captures the input provided to your model, the output it generated, and associated metadata such as timing, tokens and costs. Inference results are the fundamental units that get evaluated to assess your model's performance.

## Key Characteristics

- **Individual outputs**: Each inference result represents one model invocation
- **Rich metadata**: Includes timing, tokens, cost, and other relevant information
- **Groupable**: Can be organized into [sessions](/concepts/product/session) for related interactions
- **Evaluable**: Can be assessed using [evaluation tasks](/concepts/product/evaluation/task)

## Inference Result Properties

<ResponseField name="id" type="string" required>
  Unique identifier for the inference result.
</ResponseField>

<ResponseField name="session_id" type="string" required>
  Optional ID linking this result to a [session](/concepts/product/session) for grouping related results.
</ResponseField>

<ResponseField name="input" type="string" required>
  The input provided to the model.
</ResponseField>

<ResponseField name="actual_output" type="string" required>
  The output generated by the model.
</ResponseField>

<ResponseField name="retrieval_context" type="string" optional>
  The context retrieved by your RAG system that was used to generate the output.
</ResponseField>

<ResponseField name="latency" type="float" optional>
  Time elapsed (in milliseconds) from request to response.
</ResponseField>

<ResponseField name="usage_info" type="dict" optional>
  Token usage information for the model call.
  
  Possible keys:
  - `input_tokens`: Number of input tokens
  - `output_tokens`: Number of output tokens
  - `cache_read_input_tokens`: Number of cached input tokens
</ResponseField>

<ResponseField name="cost_info" type="dict" optional>
  Cost information for the model call.
  
  Possible keys:
  - `cost_per_input_token`: Cost per input token
  - `cost_per_output_token`: Cost per output token
  - `cost_per_cache_read_input_token`: Cost per cached input token
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Timestamp when the inference result was created.
</ResponseField>

## Lifecycle

1. **Generation**: Your AI model processes an input and generates an output
2. **Capture**: The inference result is recorded with all relevant metadata
3. **Grouping**: Optionally assigned to a [session](/concepts/product/session) for related results
4. **Evaluation**: [Evaluation tasks](/concepts/product/evaluation/task) assess the quality of the result
5. **Analysis**: Results are analyzed through the dashboard for insights and improvements

## Use Cases

### Single Inference
```
Input: "What is the capital of France?"
Output: "The capital of France is Paris."
Context: Geography knowledge base
```

### Conversational Turn
```
Session: Customer Support Chat
Input: "I need help with my order"
Output: "I'd be happy to help! Can you provide your order number?"
Previous turns: [conversation history]
```

### RAG System
```
Input: "Explain quantum computing"
Output: "Quantum computing is a type of computation..."
Retrieval Context: [relevant documents from knowledge base]
```

## Evaluation

Inference results can be evaluated in multiple ways:

- **Individual evaluation**: Assess each inference result independently
- **Session-based evaluation**: Evaluate all inference results within a session as a cohesive unit
- **Batch evaluation**: Evaluate multiple inference results across different sessions

<Info>
  [Evaluation tasks](/concepts/product/evaluation/task) use inference results as their input to generate scores and insights about your model's performance.
</Info>

## Integration with Sessions

When inference results are part of a multi-turn interaction or related workflow, they can be grouped into [sessions](/concepts/product/session):

```
Session: "User Onboarding Flow"
├── Inference Result 1: Welcome message generation
├── Inference Result 2: Feature explanation
├── Inference Result 3: FAQ response
└── Inference Result 4: Next steps recommendation
```

This grouping enables:
- **Contextual evaluation**: Assess how well the entire flow works together
- **Conversation quality**: Measure coherence and consistency across turns
- **User experience**: Analyze complete interaction patterns
